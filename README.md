## ⚙️ Investigation Workflow

Every case study follows the standard Incident Response lifecycle to ensure a systematic, repeatable, and highly documented analysis process.

| Phase | Action | Description |
| :---: | :--- | :--- |
| **1** | **Analyze** | Triage the initial alert and establish a contextual baseline. |
| **2** | **Pivot** | Correlate logs across the network to reconstruct attacker activity. |
| **3** | **Conclude** | Classify the event definitively as a True or False Positive. |
| **4** | **Remediate** | Document defensive actions, containment strategies, and follow-up. |

## 🛠️ Analyst Toolkit

> **Operational Security Notice:** All malicious Indicators of Compromise (IOCs) documented in this repository, including IPs and URLs, are professionally **defanged** (e.g., `hxxp://example[.]com`) to prevent accidental execution or navigation.

* **Platforms:** Splunk, Elastic (ELK) Stack, TryHackMe
* **Techniques:** Log Correlation, Phishing Triage, Network Traffic Analysis (NTA), Root Cause Analysis
